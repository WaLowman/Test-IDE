| Git command                                    | Purpose                                                                                           | How to instruct the IDE agent in simple natural language                      | How to do it in VS Code                                                                                                                                                                            |
| ---------------------------------------------- | ------------------------------------------------------------------------------------------------- | ----------------------------------------------------------------------------- | -------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `git clone <repo-url>`                       | Copy a remote repository to your local machine so you can work on it.                             | “Clone this repository into my workspace.”                                  | Open the Command Palette, run**Git: Clone** , paste the repository URL, then choose a local folder.                                                                                          |
| `git status`                                 | Show the current state of your working tree, including changed, staged, and untracked files.      | “Show me what has changed in this repo.”                                    | Open the**Source Control**panel. Changed files appear there automatically.                                                                                                                   |
| `git switch -c <branch-name>`                | Create a new branch and switch to it.                                                             | “Create a new branch called `feature/login-page`and switch me to it.”     | Click the current branch name in the status bar, then choose**Create new branch**and enter the name.                                                                                         |
| `git add <file>`                             | Stage changes so they are included in the next commit.                                            | “Stage `app.py`for commit.”                                               | In the**Source Control**panel, hover over the file and click the**+**icon.                                                                                                                   |
| `git add .`                                  | Stage all current changes in the working directory.                                               | “Stage all my current changes.”                                             | In the**Source Control**panel, click the**+**next to**Changes**to stage everything.                                                                                                    |
| `git commit -m "message"`                    | Save the staged changes as a new commit in local history.                                         | “Commit the staged changes with the message `Add login form validation`.” | Type a commit message in the Source Control message box, then click**Commit**or press**Ctrl+Enter**/ **Cmd+Enter** .                                                             |
| `git pull`                                   | Fetch changes from the remote and integrate them into your current branch.                        | “Pull the latest changes from the remote into my current branch.”           | Open the Source Control**...**menu and choose**Pull** , or click the sync icon if shown.                                                                                                     |
| `git push`                                   | Send your local commits to the remote repository.                                                 | “Push my current branch to the remote.”                                     | Open the Source Control**...**menu and choose**Push** , or use the sync icon.                                                                                                                |
| `git push -u origin <branch-name>`           | Push a new branch and set its upstream tracking branch so future pushes are simpler.              | “Push this new branch to origin and set it as the upstream branch.”         | Push the new branch from Source Control when prompted. VS Code usually offers to publish the branch and set upstream automatically.                                                                |
| `git diff`                                   | Show what changed. By default, it shows unstaged changes.                                         | “Show me the exact code changes I’ve made.”                                | In the**Source Control**panel, click any changed file to open the diff view.                                                                                                                 |
| `git diff --staged`                          | Show only the changes that are currently staged for the next commit.                              | “Show me what is staged for commit.”                                        | In**Source Control** , click a file under the**Staged Changes**section to view its diff.                                                                                               |
| `git log`                                    | Show commit history.                                                                              | “Show me the recent commit history.”                                        | Open the Command Palette and run**Git: View History**if available, or use the**Timeline**view on a file or repo. Many teams also use the GitLens extension for a clearer history view. |
| `git log --oneline --graph --decorate --all` | Show a compact visual history of branches and commits.                                            | “Show me a compact visual history of this repository.”                      | Use the**Timeline**view, or install**GitLens**and open its visual commit graph. VS Code’s built-in UI is limited here.                                                                |
| `git restore <file>`                         | Discard unstaged changes in a file and restore it from the last committed version or index.       | “Discard my unstaged changes in `app.py`.”                                | In**Source Control** , right-click the file and choose **Discard Changes** .                                                                                                           |
| `git restore --staged <file>`                | Unstage a file while keeping the working changes.                                                 | “Unstage `app.py`, but keep my edits.”                                    | In**Source Control** , under **Staged Changes** , hover over the file and click the**-**icon, or right-click and choose **Unstage Changes** .                                    |
| `git stash`                                  | Temporarily save uncommitted changes so you can switch context safely.                            | “Stash my current work so I can switch branches.”                           | Open the Command Palette and run**Git: Stash** .                                                                                                                                             |
| `git stash pop`                              | Reapply the most recently stashed changes and remove them from the stash list.                    | “Restore my most recently stashed work.”                                    | Open the Command Palette and run**Git: Stash Pop** .                                                                                                                                         |
| `git merge <branch-name>`                    | Combine another branch into your current branch.                                                  | “Merge `feature/login-page`into my current branch.”                       | Switch to the target branch first, then open the Command Palette and run**Git: Merge Branch...**                                                                                             |
| `git switch <branch-name>`                   | Move from one existing branch to another.                                                         | “Switch me to the `main`branch.”                                          | Click the branch name in the status bar and select the branch you want.                                                                                                                            |
| `git branch`                                 | List local branches, or manage branches. Useful for seeing where you are and what branches exist. | “Show me the branches in this repo.”                                        | Click the branch name in the status bar to see the available branches.                                                                                                                             |
| `git fetch`                                  | Download updates from the remote without merging them into your branch yet.                       | “Fetch the latest remote changes without applying them.”                    | Open the Source Control**...**menu and choose**Fetch** .                                                                                                                                     |
| `git rm <file>`                              | Remove a file from the repository and stage that deletion.                                        | “Delete `old_config.yml`and stage that deletion.”                         | Delete the file in the Explorer. VS Code will show the deletion in Source Control, where it can be committed.                                                                                      |
| `git mv <old> <new>`                         | Rename or move a file in a Git-aware way.                                                         | “Rename `draft.md`to `README.md`.”                                      | Rename or move the file in the Explorer. Git usually detects this automatically.                                                                                                                   |
| `git remote -v`                              | Show the remote repositories configured for this local repo.                                      | “Show me which remote repositories this repo points to.”                    | This is usually done in the terminal. VS Code does not expose remotes as clearly in the main Git UI.                                                                                               |
| `git init`                                   | Start a brand new Git repository in the current folder.                                           | “Initialize this folder as a new Git repository.”                           | Open the folder in VS Code, then click**Initialize Repository**in the Source Control panel.                                                                                                  |
| `git commit --amend`                         | Update the most recent commit, often to fix the message or include forgotten staged changes.      | “Amend my last commit to include the staged changes.”                       | Use the Command Palette and run**Git: Commit Staged (Amend)** , or use the Source Control**...**menu if available.                                                                           |
| `git rebase <branch-name>`                   | Replay your branch’s commits on top of another branch to create a cleaner linear history.        | “Rebase my current branch onto `main`.”                                   | This is often easiest in the terminal. Some setups expose rebase commands through the Command Palette or extensions like GitLens.                                                                  |
| `git help <command>`                         | Open help for a Git command.                                                                      | “Show me the help for `git rebase`.”                                      | This is generally done in the integrated terminal, for example with `git help rebase`.                                                                                                           |
