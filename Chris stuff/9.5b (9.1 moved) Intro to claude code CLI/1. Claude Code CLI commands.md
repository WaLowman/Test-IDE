## Claude Code CLI — 101 Guide for Professional Business Use

*Structured for IDE-based professionals, prioritised by daily utility*

### Section 1: Project Context & Memory (CLAUDE.md)

*The single highest-leverage configuration in Claude Code*

* `/init` [scans the project and auto-generates a `CLAUDE.md` with architecture and context]
* `/memory` [opens and edits your `CLAUDE.md` files and auto-memory inline]
* `CLAUDE.md` at project root [auto-loaded every session — define build commands, code standards, architecture notes]
* `~/.claude/CLAUDE.md` (Mac/Linux)
  * `%USERPROFILE%\.claude\CLAUDE.md` (Windows) [global personal instructions applied across all projects]
* `.claude/rules/*.md` [path-specific rules that only activate when Claude touches matching files]
* `@filename` syntax inside CLAUDE.md [imports the contents of another file — e.g. `@README.md`, `@package.json`]

### Section 2: Core In-Session Slash Commands

*The commands you'll use every single working day*

* `/plan` [enters read-only mode — Claude explores and plans before touching any file]
* `/compact [focus]` [summarises the conversation to free up context, optionally focused on a topic]
* `/clear` [wipes conversation history entirely and resets the context window]
* `/diff` [shows an interactive diff of all changes made in the current session]
* `/rewind` [rolls back to a previous checkpoint — can rewind code and conversation independently]
* `/cost` [shows token usage and estimated API cost for the current session]
* `/context` [visualises what is consuming your context window]
* `/model [name]` [switches model mid-session — sonnet, opus, or haiku]
* `/effort [level]` [sets reasoning depth — low, medium, high, or max]

### Section 4: MCP Server Integration

*Connecting Claude Code to your existing toolchain*

* `claude mcp add` [interactive wizard for adding an MCP server]
* `/mcp` [manage, enable, or disable MCP server connections in-session]
* `.mcp.json` at project root [project-level MCP config — define GitHub, Postgres, Jira, Slack, etc.]
* `~/.claude/.mcp.json` (Mac/Linux)
  * /`%USERPROFILE%\.claude\.mcp.json` (Windows) [personal global MCP config across all projects]
* `ENABLE_TOOL_SEARCH=auto:5` [auto-defers tool definitions when they exceed 5% of context — prevents bloat]

### Section 5: Custom Skills & Subagents

*Extending Claude Code with team-specific commands and specialist agents*

* `.claude/skills/[name]/SKILL.md` [creates a custom `/name` slash command for reusable prompts]
* `~/.claude/skills/` (Mac/Linux)
  * `%USERPROFILE%\.claude\skills\` (Windows) [personal skills available across all projects]
* `$ARGUMENTS` / `$0` in skill files [passes arguments from the command invocation into the skill]
* `.claude/agents/[name]/AGENT.md` [defines a specialist subagent — e.g. a dedicated code reviewer]
* `@agent-name do this task` [invokes a named subagent directly mid-session]
* `context: fork` in skill frontmatter [runs a skill in an isolated context with no conversation history]

### Section 6: Diagnostics & Session Insight

*Understanding what's happening under the hood*

* `/doctor` [diagnoses installation, auth, settings, and MCP issues]
* `/insights` [generates a detailed session analysis report — efficiency patterns, token habits, suggestions]
* `/stats` [shows usage history, daily streaks, and session breakdowns]
* `/status` [quick view of version, current model, and account info]
* `/usage` [shows plan usage and current rate limits]
* `Ctrl+S` in `/stats` [takes a usage screenshot — 16x faster since recent update]

### Section 7: Permission & Safety Controls

*Controlling what Claude can and cannot do autonomously*

* `Shift+Tab` [cycles between permission modes: default → plan → acceptEdits]
* `/permissions` [view and manage which tools are currently allowed]
* `--allowedTools "Read,Bash(git *)"` [pre-approves specific tool patterns at launch]
* `.claude/settings.json` allow/deny rules [project-level permission policy — committable to git]
* `--permission-mode auto` [AI classifier decides permissions per call — for trusted projects with clear CLAUDE.md rules]
* `--max-budget-usd 5.00` [hard spending cap on a session or script run]
* `--max-turns 3` [limits agentic turns — keeps automated runs from going wide]

### Section 8: Less Commonly Used (Optional Reference)

* `/fast` [toggles fast mode — same Opus 4.6 model at 2.5x speed, higher cost]
* `/loop [interval] [prompt]` [runs a prompt on a recurring schedule]
* `/schedule` [creates and manages cron-style remote agent tasks]
* `/batch [task]` [large-scale changes across 5–30 parallel git worktrees]
* `/voice` [push-to-talk voice dictation — hold Space]
* `/desktop` [hands off the current terminal session to the Desktop app]
* `/teleport` [pulls a web or iOS session into your terminal]
* `/vim` [toggles vim keybinding mode]
* `/theme` [change the colour theme]
* `/release-notes` [view the changelog for the current installed version]
* `/bug` [submit a bug report with session logs attached]
* `--chrome` [enables Chrome browser debugging integration]
* `claude -w feature-branch --tmux` [opens worktree with tmux panes for multi-pane parallel work]

### Section 9: Launching & Session Management

*How you start and control sessions from the terminal*

* `claude` [opens an interactive session]
* `claude "your prompt"` [starts a session with an immediate task]
* `claude -c` [resumes your most recent session]
* `claude -r "session-name"` [resumes a named session by its label]
* `claude -n "session-name"` [names the current session for easy recall later]
* `claude -w feature-branch` [runs Claude in an isolated git worktree — safe parallel work]
* `claude --from-pr 42` [resumes or opens a session tied to a specific GitHub PR]
* `claude -p "your prompt"` [non-interactive / print mode — processes prompt and exits, ideal for scripts and pipes]

### Section 10: Code Quality & Review Commands

*Built-in tools for professional code standards*

* `/security-review` [scans pending changes for security vulnerabilities before committing]
* `/simplify` [reviews changed code for quality issues and suggests improvements]
* `/pr-comments [PR]` [fetches GitHub PR review comments and addresses them directly]
* `/debug [description]` [structured troubleshooting within the current session]
* `git diff main | claude -p "review for security issues"` [pipes a diff for non-interactive review — scriptable]

### Section 11: Automation & CI/CD

*Running Claude Code non-interactively in pipelines*

* `claude -p "prompt"` [core non-interactive flag — processes and exits]
* `cat error.log | claude -p "what caused this crash?"` [pipe any input directly into Claude]
* `--output-format json` [returns structured JSON output for downstream parsing]
* `--json-schema '{...}'` [validates and shapes the JSON output against a schema]
* `--model haiku` [use the cheapest model for simple CI tasks like lint checks]
* `--fallback-model haiku` [falls back to a cheaper model if the primary is overloaded]
* `--bare` [skips hooks, plugins, and auto-memory — clean, deterministic runs for CI]

### Section 12: Hooks & Automation Triggers

*Running shell commands automatically on Claude Code events*

* `PostToolUse` hook [triggers after every file edit — e.g. auto-run Prettier after every write]
* `PreToolUse` hook [triggers before a tool executes — use to block or log certain actions]
* `Notification` hook [fires on Claude notifications — use for desktop alerts or Slack pings]
* `/hooks` [view and manage configured hooks in-session]
* `.claude/settings.json` hooks block [where all hooks are defined — committable to git]

### Section 13: Keyboard Shortcuts

*Speed up daily interactions*

* `Shift+Tab` [cycle permission mode — the most-used shortcut for professional work]
* `Esc + Esc` [rewind/undo to the previous checkpoint]
* `Ctrl+C` [cancel current generation]
* `Ctrl+L` [clear terminal screen and force full redraw]
* `Ctrl+G` (Mac/Linux) / `Ctrl+G` (Windows) [open current input in your external editor — vim, VS Code, etc.]
* `Option+P` (Mac)
* `	Alt+P` (Windows/Linux) [switch model without leaving the session]
* `Option+T` (Mac)
* `	Alt+T` (Windows/Linux) [toggle extended thinking]
* `!command` prefix [run a shell command directly, bypassing Claude's conversational layer]
* `@filename` prefix [autocomplete and reference a file path in your prompt]
