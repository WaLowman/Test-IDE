# NEW COMPONENTS

**The Cross-Platform Building Blocks That Have Emerged Since Mid-2025**

## Core Argument

Since roughly the middle of **2025**, the most important shift has not only been better models. It has been the rise of reusable **components** that can be mixed across different agentic systems.

These components are increasingly **cross-platform**. They do not belong to only one app or one vendor. Instead, they are becoming the new tools of the trade across Claude, ChatGPT, Gemini, and related ecosystems.

## The Big Point

The new agent stack is becoming modular.

Instead of one monolithic chatbot, we now assemble working systems out of parts such as:

- MCP servers.
- Connectors.
- Extensions.
- Plugins.
- Skills.

Each of these components solves a different problem in the agent stack.

## MCP Servers

**What they are:** the standardised tool-and-context layer for AI systems.

Anthropic describes MCP as an open protocol that standardises how applications provide context to LLMs. That is why MCP matters so much: it is trying to become the **USB-C of agent tooling**.

Why it matters:

- It creates a common protocol.
- It reduces bespoke integrations.
- It lets one capability work across multiple agent surfaces.

Cross-platform examples:

- **Anthropic** supports MCP in Claude Code, Claude Desktop, Claude.ai, and the API.
- **OpenAI** supports remote MCP-style access alongside managed connectors in the Responses API.
- **Google** now supports local and remote MCP servers in Gemini CLI workflows.

So MCP is not just "an Anthropic thing" anymore. It is becoming part of the common agent infrastructure.

## Connectors

**What they are:** managed connections to external services such as Google Drive, Gmail, Outlook, Teams, SharePoint, Slack, and Dropbox.

If MCP is the protocol layer, connectors are the **prebuilt access layer**.

Why they matter:

- They connect agents to real work systems.
- They reduce setup friction.
- They make enterprise context usable without custom code.

Cross-platform examples:

- **OpenAI** documents built-in connectors in the Responses API for services such as Dropbox, Gmail, Google Drive, Teams, Outlook, and SharePoint.
- **Anthropic** uses connectors heavily in Cowork plugins and Claude app workflows.

This is a major change from the older chatbot world, where the model mostly waited for the user to paste context manually.

## Extensions

**What they are:** installable add-ons that extend a CLI, browser, IDE, or app with new capabilities.

Extensions matter because they make the agent environment customisable without rebuilding the whole product.

Why they matter:

- They let teams add workflow-specific capabilities quickly.
- They create ecosystems around the core tool.
- They make agents feel less fixed and more programmable.

Cross-platform examples:

- Editor extensions such as Claude Code in IDEs turn the coding environment into an agentic surface.
- **Google** now treats Gemini Code Assist across VS Code and JetBrains as a core agentic coding surface.
- Browser extensions such as Claude in Chrome extend agentic behaviour into the web layer.

So extensions are the customisation layer that sits close to the user surface.

## Plugins

**What they are:** higher-level bundles that package multiple components together.

A plugin is bigger than a single connector or single skill. In Anthropic's Cowork docs, a plugin can bundle:

- Skills.
- Connectors.
- Slash commands.
- Sub-agents.

Why they matter:

- They turn many moving parts into one installable package.
- They let organisations standardise workflows.
- They make specialised agent setups easier to distribute.

Cross-platform meaning:

Even where the exact packaging differs by vendor, the pattern is spreading: users no longer want to install ten separate parts by hand. They want a packaged capability stack.

## Skills

**What they are:** portable packages of instructions, scripts, and resources that teach an agent how to perform repeatable tasks.

Anthropic introduced Agent Skills in **October 2025** and then expanded them in **December 2025** with organisation-wide management, a partner directory, and an open standard for cross-platform portability.

Why they matter:

- They package expertise.
- They reduce prompt repetition.
- They make workflows more reusable.
- They move the ecosystem from prompting to operational knowledge.

Cross-platform significance:

- Anthropic explicitly positioned skills as portable across Claude apps, Claude Code, and the API.
- The broader market is moving toward the same idea even when it uses different labels.

This is one of the clearest signs that agent systems are becoming modular and shareable.

## What Has Changed Since Mid-2025

The key change is this:

Before, we mostly talked about **models** and **chat apps**.

Now, we also have to talk about **components**.

That means the new question is no longer only:

"Which model should I use?"

It is also:

- Which MCP servers should I connect?
- Which connectors should I trust?
- Which extensions should I install?
- Which plugins should I standardise?
- Which skills should I build and share?

## Key Takeaway

This is why these components matter going forward. They are the new "tools of the trade" because they make agentic systems:

- More modular.
- More portable.
- More composable.
- More governable.
- More cross-platform.
