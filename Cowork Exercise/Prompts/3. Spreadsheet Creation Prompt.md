# SPREADSHEET CREATION PROMPT

## Role

You are a financial modelling expert who builds clean, auditable Excel financial models. You design models where every calculated value traces back to a clearly labelled assumption, so that users can test scenarios by changing inputs on a single assumptions sheet without touching any formulas.

## Context

This prompt is the final step in a three-stage workflow for Highveld-GTS's new line of business: *Authentic Cultural Experiences*:

1. **BMC Creator Prompt** — produced the Business Model Canvas Report
2. **Key Information Extraction Prompt** — extracted financial parameters from the BMC into the attached document
3. **This prompt** — builds the Excel financial model from those parameters

The attached parameters document contains revenue assumptions, cost assumptions, capital requirements, key financial assumptions, sensitivity drivers, and any gaps flagged as `⚠️ RESEARCH REQUIRED`.

## Task

Create an **Excel spreadsheet** for this new LOB with a **5-year forecast horizon**.

The objective is to assess the viability of this business from a **cash flow perspective**. Full income statements and balance sheets can be prepared later — this model focuses on whether the business generates positive cash flow and under what conditions.

**Important:** Although this is a business line extension (existing fleet, drivers, office space, etc. are already in place), build the spreadsheet as if it is a **standalone business**. Do not assume shared infrastructure at this stage.

## Instructions

**Pre-check:** Before proceeding, confirm that you have received and can read the attached **Financial Model Input Parameters** document. If it is not present, stop and notify me immediately.

### Sheet structure

Build the workbook with the following sheets, in this order:

#### Sheet 1: Model Overview
- Purpose of the model, key definitions, and a navigation guide to each sheet.
- State clearly that all inputs live on the Assumptions sheet and that no hardcoded numbers appear in any other sheet.

#### Sheet 2: Assumptions (THE CONTROL PANEL)

This is the most critical sheet in the model. Design it so that a non-technical user can open this single sheet, change any value, and immediately see the impact flow through to the Cashflow and Scenario sheets. Specifically:

1. **Layout** — Group assumptions into clearly labelled sections with visual separation (headers, borders, shading):
   - Revenue assumptions (pricing, volumes, growth rates, seasonality)
   - Cost assumptions (fixed costs, variable costs, unit economics)
   - Capital expenditure assumptions
   - Macro assumptions (inflation, discount rate, tax rate)
   - Scenario toggles (best case / base case / worst case multipliers)

2. **Every cell that the user can change** must be visually distinct — use a consistent input formatting style (e.g., light blue fill, unlocked cells) so it is immediately obvious what is an input vs. what is a formula.

3. **Research-required assumptions** — Where the attached parameters document flags values as estimated, low-confidence, or missing, populate the Value cell (Column C) with the best available estimate from the parameters document and set the Source/rationale cell (Column E) to `⚠️ RESEARCH REQUIRED`. After the model is built, the user will use Claude for Excel to research and replace these placeholder values with grounded, sourced figures (e.g., Stats SA tourism data, Gauteng Tourism Authority forecasts, industry benchmarks). **Do not perform any web research during spreadsheet creation.**

4. **Claude for Excel compatibility** — Structure the assumptions sheet so that each assumption is in its own clearly named row with:
   - Column A: Category
   - Column B: Assumption name (descriptive, plain English)
   - Column C: Value (the input cell)
   - Column D: Unit (%, ZAR, count, etc.)
   - Column E: Source / rationale
   - Column F: Confidence (High / Medium / Low)

   This flat, labelled structure makes it easy for a user working with Claude for Excel to say things like *"What happens if I increase the growth rate to 15%?"* or *"Set all Low-confidence assumptions to their worst-case values"* — and Claude can find and modify the right cells.

5. **Named ranges** — Give every assumption cell an Excel named range (e.g., `revenue_growth_rate`, `avg_ticket_price`) so that formulas in other sheets read like English and so Claude for Excel can reference them by name.

#### Sheet 3: Revenue Build-up
- Quarterly detail for all 5 years (Q1–Q4 × 5 years = 20 periods).
- Every number must reference an assumption from Sheet 2 via named ranges — no hardcoded values.
- Break revenue down by stream/product line if the parameters document identifies multiple streams.

#### Sheet 4: Cost Build-up
- Same time granularity as the Revenue sheet.
- Separate fixed and variable costs clearly.
- Variable costs must scale with the revenue drivers from Sheet 3.
- All cost values reference Sheet 2 assumptions.

#### Sheet 5: Cashflow Forecast
- Consolidate Revenue (Sheet 3) minus Costs (Sheet 4) minus Capex (from Sheet 2 assumptions).
- Show cumulative cash position.
- Highlight the breakeven point (the quarter where cumulative cash flow turns positive).
- Include a simple NPV and IRR calculation at the bottom, referencing the discount rate from Sheet 2.

#### Sheet 6: Scenario Analysis
- Provide three pre-built scenarios: **Best Case**, **Base Case**, **Worst Case**.
- Each scenario should define a set of multipliers/overrides that apply to the assumptions on Sheet 2 (e.g., Best Case = revenue +20%, costs -10%).
- Include a scenario selector (a single cell dropdown) that switches the entire model between scenarios.
- Display a side-by-side summary comparing the three scenarios on key metrics: total revenue, total costs, net cash flow, breakeven timing, NPV, and IRR.
- Make sure the scenario logic in Sheets 3–5 is wired to respond to the scenario selector — do not build three separate copies of the model.

### Design principles

- **Single source of truth:** Every input lives on Sheet 2. No magic numbers anywhere else.
- **Auditability:** A user should be able to click on any cell in Sheets 3–6 and trace it back to an assumption.
- **Experimentation-friendly:** The model must survive a user changing any single assumption without breaking. Use error handling (IFERROR) on key formulas.

## Output

Deliver the completed spreadsheet as `YYMMDD-Highveld-GTS-Financial-Model.xlsx`. Before finalising, verify:

- [ ] All formulas reference named ranges from the Assumptions sheet
- [ ] No hardcoded numbers exist outside the Assumptions sheet
