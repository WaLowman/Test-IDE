# KANBAN BOARD CREATION PROMPT

## Role

You are a Senior Front-End Developer specialising in building interactive, stateful web applications. You have deep experience with Kanban board UIs, drag-and-drop interfaces, and client-side persistence using `localStorage`.

## Context

We are working on a project for **Highveld-GTS**, a tourism services company launching an "Authentic Cultural Experiences" business line. We have already created a BMC Report and extracted the **Project Parameters** (Work Packages, Roles, Milestones, Risks) needed for execution. The Kanban methodology is the best fit to turn these parameters into a visual, actionable **Project Plan**.

This application will be built and run locally using the **Claude Code extension in VS Code**. There is no cloud database — persistence must be handled entirely on the client side using `localStorage`.

## Task

Build a complete, interactive Kanban Board as a **standalone web application** (HTML/CSS/JS or React) that I can open via `index.html` in a browser. Pre-populate it with the tasks and roles from the attached **Project Parameters** document.

## Instructions

**Pre-check:** A markdown file (**Project Parameters**) should be attached to this prompt. **If it is NOT attached, stop and warn me before proceeding.**

1. Read and analyse the attached **Project Parameters** document to identify all Work Packages, Roles, Milestones, and Risks.
2. Create a new folder called `kanban-board` inside the [4. Course Materials/Module 10/10.2 IDE Exercise/Output] folder. All application files must go inside this folder.
3. Build the Kanban Board application with an `index.html` entry point. You may use:
   - A single-page HTML file with embedded CSS and JavaScript, **or**
   - A lightweight React setup (e.g., via CDN-loaded React + Babel) — no build tools or `npm install` required.
4. Pre-populate the board with **Cards** derived from the Project Parameters, distributed across the appropriate columns.
5. Implement **drag-and-drop** functionality so users can move Cards between columns.
6. Implement **client-side persistence** using `localStorage`:
   - Save the full board state (columns, cards, positions, card details) to `localStorage` on every change.
   - On page load, restore the board from `localStorage` if saved state exists; otherwise load the default pre-populated board.
7. Implement a **Card detail modal** that opens when a Card is clicked, allowing the user to view and edit:
   - Title, Description, Assigned Role, Category, Estimated Effort, Priority, and Due Date.
8. After building, open the `kanban-board/index.html` file in the browser so I can verify it works.

## Output

The Kanban Board application must have the following characteristics:

- **Location:** All files in a `kanban-board/` folder within the [4. Course Materials/Module 10/10.2 IDE Exercise/Output] folder, with `index.html` as the entry point.
- **Columns:** Standard Kanban columns: `Backlog`, `To Do`, `In Progress`, `Review`, `Done`.
- **Pre-populated Cards:** Work Packages from the Project Parameters, each assigned to the appropriate starting column and tagged with its Category (Marketing, Operations, Tech, etc.).
- **Add Cards:** A button or mechanism to create new Cards and place them in any column.
- **Move Cards:** Drag-and-drop Cards between columns; the new position persists immediately.
- **Card Detail View:** Clicking a Card opens a modal/panel displaying and allowing edits to: Title, Description, Assigned Role, Category, Estimated Effort (days), Priority (Low / Medium / High), and Due Date.
- **Risk Log Panel:** A dedicated panel or tab listing all project risks from the Project Parameters, with columns for Risk Description, Potential Impact (High / Medium / Low), Likelihood (High / Medium / Low), and Mitigation Strategy. Users must be able to add, edit, and remove risk entries.
- **Milestone Tracker:** A dedicated panel or tab listing all project milestones from the Project Parameters. Each milestone can be linked to one or more Kanban Cards; a milestone shows as complete when all its linked Cards have reached the `Done` column.
- **Stateful & Persistent:** All board state — including columns, cards, risk log, and milestones — is saved to `localStorage`. Refreshing the page or closing and reopening the browser restores the board exactly as it was left.
- **Export Board:** An **"Export Board"** button that downloads the full board state — columns, cards, risk log, and milestones — as a `.json` file for backup.
- **Import Board:** An **"Import Board"** button that loads a previously exported `.json` file and restores the board from it.
- **Reset Board:** A **"Reset Board"** button (with a confirmation dialog) that clears `localStorage` and reloads the default pre-populated board.
- **Visual Design:** Clean, professional styling. Columns should have card counts. Apply the following shared colour palette consistently:
  - **Category colours** (card colour-coding): Marketing `#3B82F6`, Operations `#10B981`, Tech `#8B5CF6`, Finance `#F59E0B`, Partnerships `#06B6D4`, HR/Admin `#EC4899`.
  - **Priority colours**: High `#EF4444`, Medium `#F59E0B`, Low `#22C55E`.
  - This palette is shared with the Project Dashboard; both applications must use these exact values.
