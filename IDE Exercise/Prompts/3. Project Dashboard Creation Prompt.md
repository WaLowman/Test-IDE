# PROJECT DASHBOARD CREATION PROMPT

## Role

You are a Senior Front-End Developer specialising in data visualisation and analytics dashboards. You have deep experience with Chart.js, responsive dashboard layouts, and building lightweight reporting tools that run entirely in the browser.

## Context

We are working on a project for **Highveld-GTS**, a tourism services company launching an "Authentic Cultural Experiences" business line. We have already created a BMC Report, extracted **Project Parameters** (Work Packages, Roles, Milestones, Risks), and built an interactive **Kanban Board** web application that manages these tasks across columns (`Backlog`, `To Do`, `In Progress`, `Review`, `Done`).

The Kanban Board includes an **"Export Board"** function that downloads the full board state as a `.json` file. This exported JSON is the primary data source for the dashboard. The application will be built and run locally using the **Claude Code extension in VS Code** — no server, no build tools, no `npm install`.

## Task

Build a complete, interactive Project Dashboard as a **standalone web application** (HTML/CSS/JS) that I can open via `index.html` in a browser. The dashboard must import the Kanban Board's exported `.json` file and visualise the project's status, resource allocation, effort distribution, and risk profile.

## Instructions

**Pre-check:** A JSON file (the **Kanban Board export**) should be attached to this prompt. **If it is NOT attached, stop and warn me before proceeding.**

1. Read and analyse the attached Kanban Board export JSON to understand the full board state — columns, cards, categories, roles, effort estimates, priorities, and any risk or milestone data.
2. Create a new folder called `project-dashboard`within the [4. Course Materials/Module 10/10.2 IDE Exercise/Output] folder. All application files must go inside this folder.
3. Build the Dashboard application with an `index.html` entry point. Use a single-page HTML file with embedded CSS and JavaScript. Load **Chart.js** via CDN for charting — no build tools required.
4. Parse the imported JSON and generate all dashboard panels and charts as specified in the Output section below.
5. Implement a **"Load Board Data"** button that allows the user to import a different `.json` export file at any time, refreshing all dashboard panels with the new data.
6. Ensure the dashboard is **fully responsive** — usable on both desktop and tablet screen widths.
7. After building, open the `project-dashboard/index.html` file in the browser so I can verify it works.

## Output

The Project Dashboard application must have the following characteristics:

- **Location:** All files in a `project-dashboard/` folder within the [4. Course Materials/Module 10/10.2 IDE Exercise/Output] folder, with `index.html` as the entry point.
- **Header:** Dashboard title ("Highveld-GTS — Project Dashboard"), the date the data was loaded, and a **"Load Board Data"** button to import a new `.json` file.
- **Summary Cards:** A top row of summary cards showing key metrics at a glance: Total Tasks, Tasks Completed, Tasks In Progress, Overall Completion %, and Total Estimated Effort (days).
- **Task Distribution by Column (Stacked Bar Chart):** A stacked bar chart showing the number of tasks in each Kanban column (`Backlog`, `To Do`, `In Progress`, `Review`, `Done`), with segments colour-coded by Category.
- **Task Distribution by Category (Donut Chart):** A donut chart breaking down tasks by Category (Marketing, Operations, Tech, etc.), colour-coded to match the Kanban Board's card colours.
- **Effort Breakdown by Role (Horizontal Bar Chart):** A horizontal bar chart showing the total estimated effort (days) assigned to each Role.
- **Priority Distribution (Pie Chart):** A pie chart showing the proportion of tasks at each Priority level (Low, Medium, High).
- **Risk Heat Map:** A heat-map grid plotting each risk by Likelihood (x-axis) against Potential Impact (y-axis), both rated High / Medium / Low. Each cell lists the risks that fall into that combination. Colour intensity increases from green (Low–Low) through amber to red (High–High). Below the heat map, list each risk's Mitigation Strategy. Data is pulled from the `risks` array in the exported JSON.
- **Milestone Tracker:** A visual timeline or checklist showing each Milestone and its completion status (derived from whether its associated tasks have reached the `Done` column).
- **Visual Design:** Clean, professional styling consistent with the Kanban Board's look. Use a card-based layout with subtle shadows and consistent typography. Apply the same shared colour palette defined in the Kanban Board prompt:
  - **Category colours**: Marketing `#3B82F6`, Operations `#10B981`, Tech `#8B5CF6`, Finance `#F59E0B`, Partnerships `#06B6D4`, HR/Admin `#EC4899`.
  - **Priority colours**: High `#EF4444`, Medium `#F59E0B`, Low `#22C55E`.
