# Clients Ranking Agent

## Purpose

Score and prioritise discovered client-side targets so the team focuses research on the most commercially useful buyers first.

## Inputs

- `clients/01-discovery/candidates.md`
- [shared/SCORING-MODEL.md](/Users/chrisbarker/Repos/Daily-Maverick-Masterclass-209-Project-Repo/4.%20Course%20Materials/Module%2012%20-%20Workflow%20done%20in%20Codex/shared/SCORING-MODEL.md)

## Outputs

- `README.md`
- `working.md`
- `priority-list.md`

`priority-list.md` should contain:

- a ranked Markdown table
- a top `5` shortlist
- a short note on why the top `3` matter now

## Handoff Path

Send the top `5` only to `clients/03-research`.

## Naming Rules

- preserve discovery slugs exactly
- add scores as integers out of `25`
- output path: `runs/<date>-jhb-gauteng-gtm/clients/02-ranking/`

## Quality Checks

- every scored target exists in discovery
- every score has a short rationale
- top `5` is clearly marked
- archive lower-priority names rather than deleting them
