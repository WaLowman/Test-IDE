use this folder: 4. Course Materials/Module 12 - Workflow done in Codex

Create a lightweight but runnable Markdown-first sub-agent workflow for a Johannesburg tour/transport business selling curated cultural experiences to business, diplomatic, and international visitors in Gauteng.
The goal is to build a go-to-market pipeline that finds potential buyers, prioritises them, researches them, and drafts tailored outreach.
Build 3 pillars: Clients, Partners, and Experiences.
For each pillar, create 4 agents: Discovery, Ranking, Research, and Outreach.
Discovery must identify relevant organisations/people/providers; Ranking must score and prioritise them; Research must create usable dossiers on top prospects; Outreach must draft credible email/messages and prep notes.
Produce the folder structure, shared context docs, one Markdown brief per agent, a root operating guide, and `RUN.md`.
Each agent brief must define purpose, inputs, outputs, handoff paths, naming rules, and quality checks so the next stage can run cleanly.
Keep it natural-language and Markdown-first, but include the minimum shell/command wrapper needed so I can run the whole workflow with one terminal command.
Use dated outputs, consistent slugs, sequential dependencies, and pragmatic assumptions; simplify heavily for speed and demo clarity rather than completeness.
Do not create a big PRD/architecture process; collapse everything into one coherent workflow package I can demo immediately.
